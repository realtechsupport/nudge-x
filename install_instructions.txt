Instructions to run: (Work in your own user in VM)
------------------------------
Upload the install script to your VM.
Run the install script from root:

sudo sh setup.sh

-----------------------------
Clone repo:

git clone --no-checkout https://github.com/realtechsupport/nudge-x.git
Make sure you are in nudge-x Ex: your_username@nudge-x:~/nudge-x$
git sparse-checkout init --cone
git sparse-checkout set src data docs .env.example .gitignore README.md
git checkout main

All project data (metadata, exported captions, etc.) lives under data/.

Make sure to create your own .env file. The template is found in .env.example file
Make sure to add your_venv_name into .gitignore

MacOS/Linux:

python3 -m venv your_venv_name
source your_venv_name/bin/activate

# 1) Install lightweight CPU-only torch first (avoids large CUDA wheels)
python3 -m pip install --no-cache-dir --index-url https://download.pytorch.org/whl/cpu torch

# 2) Then install the rest of the dependencies
pip install -r requirements.txt
Windows:

python -m venv my_venv
my_venv\Scripts\activate
REM 1) Install lightweight CPU-only torch first (avoids large CUDA wheels)
python -m pip install --no-cache-dir --index-url https://download.pytorch.org/whl/cpu torch

REM 2) Then install the rest of the dependencies
pip install -r requirements.txt
Always run any file inside src. cd src

-----------------------------------------------------------------------------
Quick reference: main pipelines (run from inside 'src')
-----------------------------------------------------------------------------

1) Generate captions from images

MacOS / Linux:
python3 -m mllm.main.captions_pipeline

Windows:
python -m mllm.main.captions_pipeline

- Uses IMAGE_DIR from your .env
- Processes all RGB images in that folder or GCS prefix
- Writes results to the PostgreSQL 'captions' table and records a run_id

2) Vectorize accepted captions into Qdrant

python3 -m mllm.main.vectorization_pipeline

- Reads accepted captions from PostgreSQL
- Creates/updates the 'captions_collection' in Qdrant

3) Run RAG over captions only

python3 -m rag.rag_pipeline

- Opens an interactive prompt
- Uses Qdrant 'captions_collection' plus DeepSeek to answer questions

4) Run Agentic RAG (captions + documents)

python3 -m rag.agentic_rag

- Uses both 'captions_collection' and 'documents' collections in Qdrant

5) Ingest documents into Qdrant (for Agentic RAG)

python3 -m rag.document_ingestion path/to/file_or_directory

Exporting captions for a specific pipeline run
-----------------------------------------------------------------------------

From inside 'src':

1) List recent runs

python3 -m mllm.export_captions --list-runs

2) Export accepted captions for a given run_id

python3 -m mllm.export_captions <run_id>

By default this writes a TSV file to:

data/frontend_captions/captions_<short_id>_<YYYY-MM-DD>.tsv

-----------------------------------------------------------------------------
Database tips (PostgreSQL)
-----------------------------------------------------------------------------

Use this command in terminal to connect to the database:

psql -h localhost -U Admin1 -d captions_db
Enter password.

Useful commands inside psql:

\dt        - List tables in the current database (captions_db)
\x         - Toggle expanded display mode (more readable output)
\q         - Quit the psql session

You can run SQL commands like (always end with a semicolon):

SELECT COUNT(*) FROM captions;
SELECT * FROM caption_pipeline_runs ORDER BY created_at DESC LIMIT 5;
